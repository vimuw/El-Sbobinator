# Security Policy

## Supported versions

Security fixes are generally applied to the latest code on the default branch.

If you are reporting a vulnerability, please verify it against the latest available version before opening a report.

## Reporting a vulnerability

Please do **not** open a public GitHub issue for security-sensitive problems.

Instead, report vulnerabilities privately through GitHub security reporting features if enabled, or contact the project maintainers through private GitHub channels.

When possible, include:

- A clear description of the issue
- Steps to reproduce it
- The affected platform and version
- Potential impact
- Any suggested mitigation or fix

## Response expectations

Maintainers will make a best effort to:

- Confirm whether the report is valid
- Assess the severity and impact
- Prepare a fix when appropriate
- Coordinate a responsible disclosure if needed

Response times may vary depending on maintainer availability.
